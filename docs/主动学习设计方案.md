## 一、问题背景

### 1.1 动机

在字符分割任务中，人工标注成本高、效率低。通过主动学习策略，可以**智能选择最有价值的样本**进行标注，从而：
- 降低标注成本（只标注关键样本）
- 加速模型收敛（标注最有信息量的样本）
- 提升模型精度（聚焦难分样本）

### 1.2 核心假设

| 假设 | 说明 |
|------|------|
| 模型不确定的样本 | 模型预测概率接近0.5，说明模型对该样本缺乏信心 |
| 规则与模型分歧大的样本 | 规则和模型分割结果差异大，说明该样本存在歧义 |
| 难分样本包含更多信息 | 难分样本能帮助模型学习更复杂的模式 |

## 二、主动学习评分体系

### 2.1 综合评分公式

$$
AL_{score} = w_1 \cdot U + w_2 \cdot E + w_3 \cdot D + w_4 \cdot C + w_5 \cdot B
$$

| 分量 | 符号 | 权重 | 含义 | 重要性 |
|------|------|------|------|--------|
| 不确定性 | $U$ | 0.2 | 模型预测概率接近0.5的程度 | 中 |
| 熵 | $E$ | 0.2 | 预测概率分布的混乱程度 | 中 |
| **分歧** | **$D$** | **0.3** | **规则和模型分割的IoU差异** | **高** |
| 数量差异 | $C$ | 0.15 | 规则和模型识别的字符数量差异 | 低 |
| 边界位移 | $B$ | 0.15 | 规则和模型边界位置的平均差异 | 低 |

### 2.2 评分分量详解

#### 2.2.1 不确定性分数 $U$

**定义**：预测概率越接近0.5，不确定性越高

**公式**：
$$
U = \frac{1}{W} \sum_{i=1}^{W} 2 \cdot \min(p_i, 1-p_i)
$$

**解读**：
- $p_i$：第i列属于字符的预测概率（0-1）
- $\min(p_i, 1-p_i)$：距离0.5的距离（0-0.5）
- 乘以2归一化到[0, 1]范围
- **越高表示模型越不确定**

**示例**：
- $p_i = 0.5$ → $\min(0.5, 0.5) \times 2 = 1.0$（最大不确定）
- $p_i = 1.0$ → $\min(1.0, 0.0) \times 2 = 0.0$（确定是字符）
- $p_i = 0.0$ → $\min(0.0, 1.0) \times 2 = 0.0$（确定不是字符）

#### 2.2.2 熵分数 $E$

**定义**：预测概率分布越均匀，熵越大

**公式**：
$$
E = -\frac{1}{W} \sum_{i=1}^{W} \left( p_i \cdot \log(p_i) + (1-p_i) \cdot \log(1-p_i) \right)
$$

**解读**：
- 熵衡量概率分布的混乱程度
- 当 $p_i = 0.5$ 时，熵最大（$\log(2) \approx 0.693$）
- 当 $p_i = 0$ 或 $1$ 时，熵最小（0）
- **越高表示模型越不确定**

**归一化**：
$$
E_{norm} = \frac{E}{\log(2)}
$$

#### 2.2.3 分歧分数 $D$

**定义**：规则分割和模型分割结果的IoU差异

**公式**：
$$
D = 1 - \text{IoU}(\text{rule\_mask}, \text{model\_mask})
$$

**计算步骤**：
1. 根据规则分割结果生成二值掩码 $\text{rule\_mask}$
2. 根据模型分割结果生成二值掩码 $\text{model\_mask}$
3. 计算两个掩码的IoU
4. 用1减去IoU得到分歧分数

**解读**：
- IoU = 1 → D = 0：规则和模型完全一致（无分歧）
- IoU = 0 → D = 1：规则和模型完全不同（最大分歧）
- **越高表示分歧越大**

#### 2.2.4 数量差异分数 $C$

**定义**：规则和模型识别的字符数量差异

**公式**：
$$
C = \frac{|\text{rule\_count} - \text{model\_count}|}{\max(\text{rule\_count}, \text{model\_count}, 1)}
$$

**解读**：
- 当规则识别5个字符，模型识别3个字符时：$C = |5-3| / 5 = 0.4$
- 当规则和模型识别数量相同时：$C = 0$
- **越高表示数量差异越大**

#### 2.2.5 边界位移分数 $B$

**定义**：规则和模型边界位置的平均差异

**公式**：
$$
B = \min\left( \frac{\frac{1}{2N} \sum_{i=1}^{N} (|r_{start,i} - m_{start,i}| + |r_{end,i} - m_{end,i}|)}{\text{avg\_width}}, 1 \right)
$$

**计算步骤**：
1. 对齐规则和模型的字符区间（按位置匹配）
2. 计算对应区间的起始和结束位置差异
3. 除以平均字符宽度归一化
4. 取最小值不超过1

**解读**：
- $B = 0$：边界完全一致
- $B = 1$：边界差异达到平均字符宽度
- **越高表示边界差异越大**

## 三、优先级判定规则

### 3.1 优先级划分

| AL分数范围 | 优先级 | 颜色标记 | 说明 | 建议操作 |
|-----------|--------|----------|------|----------|
| $> 0.7$ | **极高** | 🔴 红色 | 模型极不确定且与规则严重分歧 | **必须标注** |
| $0.5 - 0.7$ | **高** | 🟠 橙色 | 有明显不确定性或分歧 | **强烈建议标注** |
| $0.3 - 0.5$ | **中** | 🟡 黄色 | 中等难度，存在一定歧义 | **选择性标注** |
| $< 0.3$ | **低** | 🟢 绿色 | 模型确定且与规则一致 | **无需标注** |

### 3.2 典型场景分析

| 场景 | 不确定性 | 分歧 | 数量差异 | AL分数 | 优先级 | 原因 |
|------|---------|------|---------|--------|--------|------|
| 正常印刷体 | 低 | 低 | 低 | < 0.3 | 低 | 规则和模型都能正确分割 |
| 粘连字符 | 高 | 高 | 高 | > 0.7 | 极高 | 规则可能切分错误，模型也不确定 |
| 模糊字符 | 高 | 中 | 低 | 0.5-0.7 | 高 | 模型不确定，但规则可能正确 |
| 特殊符号 | 中 | 高 | 中 | 0.5-0.7 | 高 | 规则和模型对符号分割有分歧 |

## 五、算法实现

### 5.1 评分计算伪代码

```python
def compute_al_score(line_id):
    # 1. 获取模型预测结果
    model_intervals, pred_prob = model.predict(line_id)
    
    # 2. 获取规则分割结果
    rule_intervals = load_rule_result(line_id)
    
    # 3. 计算各分量分数
    uncertainty = mean(min(p, 1-p) * 2 for p in pred_prob)
    entropy = mean(-p*log(p) - (1-p)*log(1-p) for p in pred_prob) / log(2)
    disagreement = 1 - iou(rule_mask, model_mask)
    count_diff = abs(len(rule_intervals) - len(model_intervals)) / max(...)
    boundary_diff = mean_boundary_displacement(rule_intervals, model_intervals)
    
    # 4. 计算综合分数
    al_score = (0.2 * uncertainty + 0.2 * entropy + 
                0.3 * disagreement + 0.15 * count_diff + 
                0.15 * boundary_diff)
    
    return al_score
```

### 5.2 IoU计算伪代码

```python
def iou(rule_mask, model_mask):
    intersection = sum(r & m for r, m in zip(rule_mask, model_mask))
    union = sum(r | m for r, m in zip(rule_mask, model_mask))
    return intersection / union if union > 0 else 0
```

### 5.3 边界位移计算伪代码

```python
def mean_boundary_displacement(rule_intervals, model_intervals):
    if len(rule_intervals) != len(model_intervals):
        return 1.0
    
    total_displacement = 0
    total_width = 0
    
    for (r_start, r_end), (m_start, m_end) in zip(rule_intervals, model_intervals):
        total_displacement += abs(r_start - m_start) + abs(r_end - m_end)
        total_width += (r_end - r_start) + (m_end - m_start)
    
    avg_width = total_width / (2 * len(rule_intervals))
    avg_displacement = total_displacement / (2 * len(rule_intervals))
    
    return min(avg_displacement / avg_width, 1.0)
```

## 十、基于规则的主动学习（v1.1新增）

### 10.1 设计动机

在没有训练好的模型时，仍需要快速识别可能切割错误的样本。基于规则切割结果的主动学习方法，直接分析规则JSON中的字符特征，无需模型推理，速度快、成本低。

### 10.2 评分体系

算法同时分析**原始切割结果**和**合并后处理结果**，综合评估切割质量。

**综合评分公式**：

$$
AL_{score}^{rule} = w_1 \cdot N + w_2 \cdot W + w_3 \cdot G + w_4 \cdot V + w_5 \cdot A + w_6 \cdot E + w_7 \cdot D + w_8 \cdot M + w_9 \cdot N_m + w_{10} \cdot W_m
$$

| 分量 | 符号 | 权重 | 含义 | 检测目标 |
|------|------|------|------|----------|
| **合并前指标** | | | | |
| 窄字符比例 | $N$ | 0.15 | width < 平均宽度×0.5 的字符 | 分割过细 |
| 宽字符比例 | $W$ | 0.15 | width > 平均宽度×2.0 的字符 | 粘连未分开 |
| 间隙异常 | $G$ | 0.1 | gap < 2px 或 gap > 平均间隙×3.0 | 分割错误 |
| 宽度变异系数 | $V$ | 0.1 | 宽度标准差/平均宽度 | 分割不稳定 |
| 极端宽高比 | $A$ | 0.08 | aspect_ratio < 0.3 或 > 3.0 | 分割错误 |
| 空白区域 | $E$ | 0.08 | 行首行尾空白占比 | 漏识别 |
| 行密度异常 | $D$ | 0.08 | 字符密度 < 0.1 或 > 0.8 | 粘连或漏识别 |
| **合并后指标** | | | | |
| 合并率 | $M$ | 0.12 | 合并前后字符数量减少比例 | 原始切割质量 |
| 合并后窄字符 | $N_m$ | 0.08 | 合并后仍然存在的窄字符比例 | 合并残留问题 |
| 合并后宽字符 | $W_m$ | 0.06 | 合并后仍然存在的宽字符比例 | 合并残留问题 |

**合并逻辑说明**：

合并后处理使用与训练时一致的规则（与 `LabelGenerator._merge_narrow_chars` 逻辑相同）：

| 合并条件 | 阈值 |
|---------|------|
| 间隙 | < 3px |
| 单字符宽高比上限 | 0.7 |
| 合并后宽高比下限 | 0.5 |
| 合并后宽高比上限 | 1.5 |

**合并率计算公式**：

$$
M = 1 - \frac{\text{合并后字符数}}{\text{合并前字符数}}
$$

合并率越高，表示原始切割产生的过细分割越多，需要合并的字符越多，切割质量越差。

### 10.3 评分分量详解

#### 10.3.1 窄字符比例 $N$

**定义**：宽度远小于平均宽度的字符比例

**公式**：
$$
N = \frac{\text{窄字符数}}{\text{总字符数}}
$$

**窄字符判定条件**：
- $width < \text{平均宽度} \times 0.5$
- $width/height < 0.5$

**解读**：窄字符通常表示分割过细，一个字符被分成了多个部分。

#### 10.3.2 宽字符比例 $W$

**定义**：宽度远大于平均宽度的字符比例

**公式**：
$$
W = \frac{\text{宽字符数}}{\text{总字符数}}
$$

**宽字符判定条件**：
- $width > \text{平均宽度} \times 2.0$ **或**
- $width/height > 2.0$

**解读**：宽字符通常表示粘连字符未分开，多个字符被合并成了一个。

#### 10.3.3 间隙异常分数 $G$

**定义**：过小或过大间隙的比例

**公式**：
$$
G = \frac{\text{异常间隙数}}{\text{总间隙数}}
$$

**异常间隙判定条件**：
- 间隙过小：$gap < 2\ \text{px}$（可能分割过细）
- 间隙过大：$gap > \text{平均间隙} \times 3.0$（可能漏分割）

#### 10.3.4 宽度变异系数 $V$

**定义**：字符宽度的变异系数（标准差/均值）

**公式**：
$$
V = \frac{\sigma_{\text{width}}}{\mu_{\text{width}}}
$$

**归一化**：$V_{norm} = \min(V, 2.0) / 2.0$

**解读**：变异系数越大，表示字符宽度变化越大，分割结果越不稳定。

#### 10.3.5 极端宽高比分数 $A$

**定义**：宽高比异常的字符比例

**公式**：
$$
A = \frac{\text{极端宽高比字符数}}{\text{总字符数}}
$$

**极端宽高比判定条件**：
- $aspect\_ratio < 0.3$（过窄）
- $aspect\_ratio > 3.0$（过宽）

#### 10.3.6 空白区域分数 $E$

**定义**：行首和行尾空白区域占图像宽度的比例

**公式**：
$$
E = \frac{\text{左侧空白} + \text{右侧空白}}{\text{图像宽度}}
$$

**解读**：空白区域过大可能表示漏识别或图像边缘问题。

#### 10.3.7 行密度异常分数 $D$

**定义**：字符总宽度占图像宽度的比例是否异常

**公式**：
$$
\text{density} = \frac{\sum \text{字符宽度}}{\text{图像宽度}}
$$

**异常判定**：
- 密度过低：$density < 0.1$（稀疏文本或漏识别）
- 密度过高：$density > 0.8$（粘连严重或分割过粗）

**得分计算**：$D = \min(|\text{density} - 0.45| / 0.45, 1.0)$

### 10.4 优先级判定规则

| AL分数范围 | 优先级 | 颜色标记 | 说明 | 建议操作 |
|-----------|--------|----------|------|----------|
| $> 0.7$ | **极高** | 🔴 红色 | 存在严重切割问题 | **必须审核** |
| $0.5 - 0.7$ | **高** | 🟠 橙色 | 存在明显异常 | **强烈建议审核** |
| $0.3 - 0.5$ | **中** | 🟡 黄色 | 存在轻微异常 | **选择性审核** |
| $< 0.3$ | **低** | 🟢 绿色 | 切割结果正常 | **无需审核** |

### 10.5 典型场景分析

| 场景 | 窄字符 | 宽字符 | 间隙异常 | AL分数 | 优先级 | 原因 |
|------|--------|--------|----------|--------|--------|------|
| 正常印刷体 | 低 | 低 | 低 | < 0.3 | 低 | 字符宽度均匀，间隙正常 |
| 粘连字符 | 低 | 高 | 低 | > 0.7 | 极高 | 多个字符被合并成一个宽字符 |
| 过细分割 | 高 | 低 | 高 | > 0.7 | 极高 | 一个字符被分成多个窄字符 |
| 混合异常 | 中 | 中 | 中 | 0.5-0.7 | 高 | 存在多种类型的切割问题 |

### 10.7 两种主动学习方式对比

| 特性 | 基于模型 (`active-learn`) | 基于规则 (`rule-based-al`) |
|------|--------------------------|---------------------------|
| 需要模型 | ✅ 是 | ❌ 否 |
| GPU加速 | ✅ 支持 | ❌ 不需要 |
| 速度 | 较慢（需推理） | 很快（仅分析JSON） |
| 适用场景 | 有训练好的模型时 | 无模型或快速筛选时 |
| 核心逻辑 | 模型预测不确定性+分歧 | 合并后特征异常检测（粘连/过度/不足） |
| 评分维度 | 5个（不确定性、熵、分歧、数量差异、边界位移） | 9个（粘连、过度合并、合并不足+6个辅助） |
| 输出文件 | `al_ranking.json` | `rule_based_al_ranking.json` |

### 10.8 组合使用策略

建议在实际应用中组合使用两种主动学习方式：

```
Step 1: 规则筛选（快速）
  └── 使用 rule-based-al 筛选出候选集（如 Top 1000）

Step 2: 模型精排（精确）
  └── 使用 active-learn 对候选集进行精排（如 Top 100）

Step 3: 人工审核
  └── 审核精排后的样本
```

**优势**：
- 规则筛选速度快，可处理全量数据
- 模型精排精度高，聚焦最有价值的样本
- 组合使用兼顾效率和精度

## 九、注意事项

### 9.1 权重调整

- 权重需要根据实际数据分布调整
- 建议通过交叉验证确定最优权重
- 不同阶段可使用不同权重配置

### 9.2 计算效率

- 对4609个行图像计算AL分数约需10-20分钟
- 可通过批量推理优化速度
- 建议离线计算，定时更新排名

### 9.3 标注质量

- 主动学习依赖高质量标注
- 建议对标注结果进行抽检
- 标注标准应保持一致

### 9.4 模型更新

- 每次标注后应重新训练模型
- 建议保留历史模型版本
- 对比不同迭代阶段的模型性能
