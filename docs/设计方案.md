## 一、原始数据链路与存储设计

### 1.1 需求摘要

原始数据源为多份PDF文档，需支持 **PDF文件 → 页面图像 → 单行文本图像 → 单字符图像** 四级分层处理与独立存储。支持留存多版本切割结果，原始图像实体仅存储一份。

### 1.2 目录结构设计

```
datahome/                              # 数据基础目录
├── raw/                               # 原始数据层（仅存储一份）
│   ├── pdf/                           # 原始PDF文件
│   │   └── gwyb195827.pdf
│   └── pages/                         # 页面图像（PDF转图片结果）
│       └── page_pdf_gwyb195827_1.png
├── lines/                             # 行图像（图片转行结果）
│   └── line_page_pdf_gwyb195827_1_0.png
├── chars/                             # 单字符图像（行转字符结果）
│   └── char_line_page_pdf_gwyb195827_1_0_0.png
├── annotations/                       # 标注数据
│   └── annotation_line_page_pdf_gwyb195827_1_0.json
├── datasets/                          # 数据集定义
│   └── dataset_train_v1.json
├── rule_jsons/                        # 规则切割结果
│   └── line_page_pdf_gwyb195827_1_0_rule.json
├── lineage/                           # 血缘索引目录（分片存储）
│   ├── pdf_gwyb195827_lineage.json    # 单个PDF的血缘信息
│   └── lineage_index.json             # 顶层汇总索引
└── lineage.json                       # 全量血缘索引（兼容旧版本）
```

### 1.3 ID命名规范

| 层级 | ID格式 | 示例 |
|------|--------|------|
| PDF | `pdf_{stem}` | `pdf_gwyb195827` |
| 页面 | `page_{pdf_id}_{page_num}` | `page_pdf_gwyb195827_1` |
| 行 | `line_{page_id}_{line_idx}` | `line_page_pdf_gwyb195827_1_0` |
| 字符 | `char_{line_id}_{char_idx}` | `char_line_page_pdf_gwyb195827_1_0_0` |

## 二、全链路可追溯设计

### 2.1 需求摘要

任意一张单字符图像、单行文本图像，均可回溯到其所属的原始PDF文件、对应页码，以及在原始页面中的像素级坐标位置。

### 2.2 追溯链路

```
字符ID → 行ID → 页面ID → PDF ID → 原始PDF文件 + 像素坐标
```

### 2.3 坐标系统

| 坐标类型 | 定义 | 示例 |
|----------|------|------|
| 相对坐标 | 相对于所在行/页面的偏移 | `col_start=10, col_end=60` |
| 绝对坐标 | 相对于原始页面左上角的像素位置 | `abs_x=10, abs_y=100, abs_x_end=60, abs_y_end=150` |

### 2.4 查询接口

| 接口 | 功能 | 参数 | 返回 |
|------|------|------|------|
| `get_char_by_id(char_id)` | 根据字符ID获取完整信息 | `char_id: str` | 字符完整信息（含绝对坐标） |
| `get_line_by_id(line_id)` | 根据行ID获取行信息 | `line_id: str` | 行完整信息 |
| `get_page_by_id(page_id)` | 根据页面ID获取页面信息 | `page_id: str` | 页面完整信息 |
| `get_pdf_by_id(pdf_id)` | 根据PDF ID获取PDF信息 | `pdf_id: str` | PDF完整信息 |
| `get_lines_by_page(page_id)` | 获取页面的所有行 | `page_id: str` | 行信息列表 |
| `get_chars_by_line(line_id)` | 获取行的所有字符 | `line_id: str` | 字符信息列表 |

### 2.5 实现状态

| 功能 | 状态 | 说明 |
|------|------|------|
| 字符追溯 | 已实现 | 可通过 char_id 获取完整链路 |
| 行追溯 | 已实现 | 可通过 line_id 获取完整链路 |
| 绝对坐标存储 | 已实现 | char_info 中存储 abs_x, abs_y |

## 四、标注数据设计

### 4.1 需求摘要

以单行文本为标注单元，支持纯列标注与矩形框/多边形标注；支持预标注、标注导出与回库。

### 4.2 标注数据结构

```json
// annotation_line_page_pdf_gwyb195827_1_0.json
{
  "line_id": "line_page_pdf_gwyb195827_1_0",
  "annotation_id": "anno_line_page_pdf_gwyb195827_1_0",
  "annotation_type": "column",
  "created_at": "2026-06-26T00:00:00",
  "updated_at": "2026-06-26T00:00:00",
  "created_by": "annotator_001",
  "status": "annotated",
  "confidence": 1.0,
  "is_pre_annotation": false,
  "source_type": "manual",
  "content": {
    "columns": [10, 60, 110, 160],
    "chars": [
      {
        "char_idx": 0,
        "col_start": 10,
        "col_end": 60,
        "label": "中"
      }
    ]
  }
}
```

### 4.3 标注类型

| 类型 | 说明 | 适用场景 |
|------|------|----------|
| `column` | 纯列标注（切割线 x 坐标） | 规则切割结果修正 |
| `bbox` | 矩形框标注 | 不规则字符检测 |
| `polygon` | 多边形标注 | 复杂排版场景 |

### 4.4 标注状态流转

```
unannotated → pre_annotated → annotated → verified → rejected
```
