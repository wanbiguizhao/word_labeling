一、原始数据链路与存储需求
原始数据源为多份 PDF 文档，需支持 PDF文件 → 页面图像 → 单行文本图像 → 单字符图像 四级分层处理与独立存储。
支持留存多版本切割结果（规则切割、不同迭代的模型切割），各版本互不覆盖。
原始图像实体仅存储一份，避免多副本冗余占用存储空间。
二、全链路可追溯需求
任意一张单字符图像、单行文本图像，均可回溯到其所属的原始 PDF 文件、对应页码，以及在原始页面中的像素级坐标位置。
三、数据集管理需求
数据集为文本行的逻辑集合，不重复复制图片实体；同一行数据可归属于多个数据集。
支持两种数据集创建方式：
整 PDF 全量导入：指定单个或多个 PDF，将其包含的所有文本行全部加入数据集。
跨 PDF 筛选导入：从指定的若干 PDF 中，按条件筛选部分文本行加入数据集。
支持多维度筛选文本行，可组合使用：标注状态、切割置信度 / 不确定性、粘连程度、标点密度、数量限制、手动指定行 ID 等。
支持数据集锁定，锁定后不可增删内容，保障标注任务、训练实验的可复现性。
支持自动统计数据集的总行数、覆盖 PDF 数量、标注完成度等核心指标。
四、标注数据需求
以单行文本为标注单元，支持两种标注格式：纯列标注（切割线 x 坐标）、矩形框 / 多边形标注。
支持基于现有规则 / 模型自动生成预标注，辅助人工修正，提升标注效率。
支持一键导出标注任务包（含待标注图片、预标注文件、任务清单），适配标注工具。
支持标注结果一键回库，自动更新对应行的标注状态与标注内容。
支持按数据集维度统计、追踪标注进度。
五、训练数据支撑需求
支持一键导出适配不同模型的标准训练数据集，包括 U-Net 列分割格式、YOLO 检测 / 分割格式。
支持按 PDF 维度拆分训练集与验证集，避免数据泄露。
支持区分真实标注数据与伪标签数据，适配半监督训练场景。
支持记录每次训练集导出的版本、筛选条件与拆分规则，保障实验可复现。
六、迭代闭环数据需求
支持主动学习采样：按不确定性、难例维度筛选样本生成标注数据集，支撑小步迭代。
支持伪标签的生成、质量过滤与属性标记，明确区分真实标注与伪标签数据。
支持错误样本、边界难例的定向沉淀与归集，用于补充标注、针对性优化模型。
支持新 PDF 入库后自动完成全链路处理与质量评估，高价值难例自动进入待标注池。
